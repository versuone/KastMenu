Command

convert

: path: /opt/kastmenu-server/current/bin/convert usage: use this command to convert to from xml, yaml, hcl.
convert -h Usage: convert convert --to[yaml|hcl|json] This command accept only one sole argument: Note: must be properly suffixed ! By default the command will convert any input file to xml. With Options: --toyaml, --tok8yaml, --tohcl or --tojson Will convert any input file yaml, hcl or json. -------------- | e.g. tests | -------------- cd /opt/kastmenu/current/bin - menu may be replaced by menu1 or menu2. # from xaml: # ---------- # --toyaml: ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.xml --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --toyaml -v300 # --tohcl ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.xml --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --tohcl -v300 # from yaml: # ---------- # --toxml: ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.yaml --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --toxml -v300 # --tohcl: ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.yaml --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --tohcl -v300 # from hcl: # --------- # --toxml: ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.hcl --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --toxml -v300 # --toyaml: ./convert -f /opt/kastmenu/current/samples/kastmenu/menu.hcl --file_desc /opt/kastmenu/current/conf/descs/menu.desc.xml --toyaml -v300 Options: -h, --help show this help message and exit -f FILE_SOURCE, --file_source=FILE_SOURCE (Optional) file_source: a file containing one or many softclasses separated by --- or ... Exclusive with using the argument and file_dir (--cacdir, -c) or STDIN pushing directly softclass source into the pipe. his file can be either: xml, hcl or yaml and jinja or mako e.g.: my.yaml.jinja or my.xml.mako. -D FILE_DESC, --file_desc=FILE_DESC A file descriptor that is used to check the input file. Allowed suffixes are: xml, swagger, json, yaml, hcl Notice: json, yaml, hcl are accepted if they were generated by convert. swagger are standard yaml OpenApi swagger files. -v VERBOSE, --verbose=VERBOSE The verbose level. --toxml (optional) parse to xml, No need this is the default. --toyaml (optional) Instead of parsing to xml, this will parse to yaml. --tok8yaml (optional) Instead of parsing to xml, this will parse to yaml. tok8yaml will withdraw the top tag. --tohcl (optional) Instead of parsing to xml, this will parse to hcl. --tojson (optional) Instead of parsing to xml, this will parse to json. --toterra (optional) Instead of parsing to xml, this will parse to an hcl file (With --desconly also provided to a json file) but with names expected by Terraform (All tag names in lower cases...). --desconly (optional) This will output the associated descriptor file as xml (default), yaml, hcl, json or terra file (depending if --toyaml, --toyhcl, --tojson, --toterra is provided). -t, --show_dft (optional) By default Attributes with value matching the default value into the descriptor are not shown ! if --show_dft (-t) is provided they would be shown. --temp_dir=TEMP_DIR Temporary directory (optional usefull for mako or jinja debug). --keep_temp_dir This will keep the temporary dir ! Allowing to see all the intermediate state will parsing the file. e.g: from mako, jinja, yaml, hcl to xml. Beware Parsing is usually done in memory, keeping the resulting files into temp_dir could be a security breach. --notcase Case sensitivity, Do we ignore tag case. -c, --checkonly Check only. --emhclcheck Internal GR check. --tmpl_kws=TMPL_KWS (optional) A set of parameters (a CoolTyped dict) to feed mako or jinja with, when the softclass file found into the file_dir rather than beeing an .xml, .yaml or .hcl file is ending by .jinja or .mako (e.g. myfile.xml.jinja or myfile.yaml.jinja). STDIN Options: When using kact from stdin. E.g.: cat my_file_source[.xml|.yaml|.hcl|. xml.jinja|.yaml.jinja|.hcl.jinja|.xml.mako|.yaml.mako...] | kact -v3, kact cannot gess the file type from the suffixe. So it must be provided. --xml (STDIN only) The type of the input combined_file is xml --hcl (STDIN only) The type of the input combined_file is hcl --yaml (STDIN only) The type of the input combined_file is yaml --mako (STDIN only) The type of the input combined_file is mako works together with --xml, --yaml or --hcl. --jinja (STDIN only) The type of the input combined_file is jinja with --xml, --yaml or --hcl.
